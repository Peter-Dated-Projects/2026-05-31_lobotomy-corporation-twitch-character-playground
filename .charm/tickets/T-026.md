Implement layers L1 (steering core) and L2 (facing + sprite flip) of the agent-behavior model. Read the curated design .charm/kb/architecture/agent-behavior-model.md + .charm/kb/decisions/0003-layered-agent-behavior.md, and the concrete code-level recommendations in docs/research/npc-movement-steering.md (sections 2A-2E + mapping table). Those are authoritative; implement what they specify.

L2 (do first, it's the visible bug): character.py tracks _facing but render/scene.py never flips the sprite, so everyone faces the sheet default direction. Expose facing as public read-only state on Character; in render/scene.py flip the frame with pygame.transform.flip(surf, True, False) when facing<0; add a WALK_THRESHOLD deadzone/hysteresis so a near-idle character (only separation push) doesn't strobe. Record in docs/design/sidescroller-contract.md which sign is the unflipped pose (Sim exposes facing, World/scene consumes it).

L1: in steering.py (keep the pure-function-over-vectors style of separation_push) add: a coherent 1D wander helper (persistent drifting heading, NOT random.choice reroll), and an arrive/steer helper (desired-velocity easing). In character.py: replace the straight-line _facing*WALK_SPEED wander + random.choice direction flips with the drifting wander heading; ease velocity toward desired via a capped steering force (wire up the currently-unused settings.MAX_SPEED; add WALK_ACCEL/MAX_FORCE) instead of snapping; apply arrive slowdown for GROUPED followers inside a GROUP_SLOW_RADIUS before the existing arrive-radius snap; make the EMOTING velocity*=0.6 decay dt-based. Add all new tunables to settings.py (WANDER_DISPLACE, WANDER_REORIENT_CHANCE, WALK_ACCEL/MAX_FORCE, GROUP_SLOW_RADIUS) grouped + commented; never hardcode inline.

Keep behaviour frame-rate-independent (dt-scaled), preserving the existing pattern. Do NOT change world.py's update signature in this ticket (that plumbing lands in the crowd ticket) — wander/easing here operate on the existing neighbor_positions. Extend tests/test_steering.py (wander heading drifts coherently, not random; arrive eases) and tests/test_character.py (facing updates with deadzone; velocity eases rather than snaps). Run `uv run --project . pytest` green. Touch only the listed files.
