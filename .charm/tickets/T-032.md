Change the window to a wide, short letterbox stage and re-tune the side-scroller layout to fit it. Currently SCREEN_W=960, SCREEN_H=540 with GROUND_TOP=SCREEN_H-60 and floating platform tiers tuned for 540px of height.

TARGET: SCREEN_W=1920 (max 1080p width), SCREEN_H=200. This is ~1/3 the previous height, so a blind constant swap breaks the level — the floating platforms and jump reachability must be re-derived for 200px.

DO:
1. settings.py: SCREEN_W=1920, SCREEN_H=200. Re-tune GROUND_TOP so the ground band sits near the bottom with characters (~40px tall + nameplate above) fully visible. Keep jump physics consistent: apex ~= JUMP_SPEED^2/(2*GRAVITY) (~140px at current values) must still let a character reach whatever floating tier you keep — either lower JUMP_SPEED/raise the single tier so it fits within 200px, or reduce to at most ONE floating tier. Whatever you choose, every platform must be reachable in a single jump from the tier below (the contract's invariant).
2. sim/platforms.py (default_level): rework the platform set for a 1920-wide, 200-tall stage — full-width ground at index 0, plus at most 1 (maybe 2 short) floating platforms placed so they're reachable and don't overlap the nameplate zone. Wider stage means platforms can be longer/more spread horizontally.
3. Verify nameplates + sprites fit: at 200px tall, a character standing on a floating platform with a nameplate above must not clip off the top. If it's too tight, note it and place the floating tier low enough that it works (or flag that nameplates need shrinking as a follow-up — do NOT edit render/hud or scene here).

VERIFY: update tests/test_platforms.py (reachability: each tier within one jump of the one below, using the actual GRAVITY/JUMP_SPEED; platforms within screen bounds) and any test_world spawn/bounds assertions that hardcode the old 960x540. Run a headless check that a character can spawn, walk the full 1920 width, and jump onto the floating tier. Full `uv run --project . pytest` green. Touch only the listed files; do not change render/scene.py, hud.py, or character.py.
