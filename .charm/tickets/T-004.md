Build the TTS engine that turns a filtered chat message into spoken audio in a chosen Sephirah's cloned voice, using KokoClone. Foundational ticket; others depend on it.

## Engine decision (already made)
See `.charm/kb/tts-engine-decision.md`. KokoClone = Kokoro-ONNX (synth in a preset voice) -> Kanade voice-conversion model (re-voice toward a reference WAV). NOT embedding-based; reference is consumed per utterance. Whole stack is commercial-safe (Kokoro Apache 2.0, Kanade MIT, KokoClone Apache 2.0).

## What to build
Create package `twitch_playground/speak/` with `__init__.py` and `tts.py`.

KokoClone is NOT on PyPI (repo github.com/Ashish-Patnaik/kokoclone, core/cloner.py). VENDOR the minimal cloner logic into `twitch_playground/speak/tts.py` or a `_kokoclone.py` helper, adapted from that repo's `core/cloner.py` + `core/chunked_convert.py`. Apache 2.0 permits this -- add a short attribution comment naming the source repo + license. Do NOT depend on a git checkout of kokoclone.

A verified spike lives at `/tmp/kokoclone-spike` (cloned repo + working venv). Read `/tmp/kokoclone-spike/core/cloner.py` for the exact `KokoClone.generate(text, lang, reference_audio, output_path)` / `.convert()` flow.

### SpeakEngine class
- `SpeakEngine(voices_dir, roster)`: load Kanade model + vocoder ONCE at construction (~25-30s). Auto-detect CUDA/CPU via torch.
- Cache the loaded reference-audio tensor PER character at startup. Reference WAVs: `twitch_playground/assets/voices/<name>.wav` (built by scripts/setup_voices.py; roster = hod, malkuth, netzach, yesod). Do NOT re-read the WAV every call.
- `synthesize(text, character) -> (samples: np.ndarray, sample_rate: int)`: Kokoro synth + Kanade VC. Pure compute, no playback.
- `speak(text, character)`: synthesize then play on a dedicated pygame mixer channel on a BACKGROUND thread (sim loop must not block seconds). Expose an is-speaking signal / callback so wiring can clear the balloon when playback ends.
- Graceful fallback: missing reference WAV or unavailable torch/models -> raise a clear typed error wiring can catch and skip (never crash the loop).
