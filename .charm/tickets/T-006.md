Research the best way to connect a Python/pygame application to Twitch chat so it can listen for specific commands from viewers.

Cover:
- Twitch IRC protocol: connecting, authenticating with OAuth, joining a channel
- twitchio library vs raw IRC socket vs websocket approaches — tradeoffs
- How to parse chat messages for a command prefix (e.g. !spawn, !character)
- Rate limits and reconnection handling
- Running the IRC listener on a background thread so it doesn't block the pygame render loop — thread-safe queue pattern to pass events to the main thread
- Twitch EventSub as an alternative to IRC for chat commands
- What OAuth scopes are needed, how to get a bot token cheaply
- Example minimal bot that reads chat and puts command events into a queue

Write your findings to research/twitch-chat.md. Deliver a research brief with a recommended approach and a minimal working code skeleton.
