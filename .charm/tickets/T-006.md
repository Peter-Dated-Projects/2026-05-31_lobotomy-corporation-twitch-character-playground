Improve reference-audio quality for cloning. Current `scripts/setup_voices.py` concatenates full dub clips into 8-11 min WAVs that still carry background music/SFX. Kanade VC wants a SHORT, CLEAN reference (~10-15s solo speech), not a long noisy one.

## What to change (only scripts/setup_voices.py)
1. After concat + resample, produce a trimmed reference: ~10-20s of the cleanest solo-speech segment. A simple deterministic heuristic is fine (e.g. fixed window offset past the intro, or an easy energy/silence pick). Document it; do not over-engineer.
2. Make the optional `--denoise` deepfilternet pass wire in cleanly (it exists as a stub that warns if deepfilternet missing). Run it on the TRIMMED clip, not the 10-min file.
3. Ensure `twitch_playground/assets/voices/<name>.wav` ends up being the trimmed/cleaned reference (the engine reads that path). Stash the long version under a different name (e.g. `<name>.full.wav`) if kept, or document.
4. Preserve existing flags (--only, --force, --list, --denoise) + roster/video-id config. Do not change download logic or character set.

## Constraints
- Only touch scripts/setup_voices.py. Output dir + gitignore already set up.
- Needs yt-dlp + ffmpeg (already used). --denoise needs deepfilternet (optional; warn-and-skip if absent).
- Idempotent + --force must still hold.

## Definition of done
- `uv run scripts/setup_voices.py --only hod --force` produces a short clean voices/hod.wav (~10-20s); works for all four.
- --list + other flags still work.
- Module docstring documents the trim heuristic + where the full clip goes.
