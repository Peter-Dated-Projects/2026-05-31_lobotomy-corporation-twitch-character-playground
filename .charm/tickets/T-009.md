Make the speak engine use the GPU (CUDA on the streaming PC's RTX 5070Ti; MPS on Apple Silicon if it works) and NOT silently fall back to CPU-only, which runs slower than real-time (RTF ~1.4). Right now the engine just picks `cuda if torch.cuda.is_available() else cpu`, and plain `torch>=2.9` from PyPI installs the CPU-only wheel on Windows/Linux -- so it is easy to end up on CPU without noticing.

This builds on the completed engine ticket T-004 (depends_on T-004). It touches ONLY the engine files + pyproject; it is disjoint from the wiring ticket (T-008) so they run in parallel. The engine should read its GPU config from an ENV VAR directly so the wiring layer needs no change.

## 1. Engine device selection (twitch_playground/speak/tts.py + _kokoclone.py)
- Device preference order: CUDA -> MPS (Apple Silicon) -> CPU. Replace the current `cuda else cpu` check.
- MPS is untested for Kanade/Kokoro and torch MPS has gaps: attempt MPS, but wrap model load/first-inference in try/except and fall back to CPU with a clear warning if it errors. Document this.
- Read env var `SPEAK_REQUIRE_GPU` (default: false). When true and neither CUDA nor MPS is available/usable, raise `SpeakUnavailableError` with a clear message ('GPU required but none available; set SPEAK_REQUIRE_GPU=0 to allow CPU'). When false, allow CPU but log a prominent one-line WARNING that synthesis will be slower than real-time.
- ALWAYS log the selected device at engine init, e.g. `[speak] device=cuda`. Make it obvious in stdout which path is active.
- Optional explicit override: honor `SPEAK_DEVICE` env (cuda|mps|cpu|auto, default auto) if easy; auto = the preference order above.
- Keep the existing typed-error contract and lazy-import behavior from T-004 intact. Do not regress the 96 passing tests.

## 2. CUDA torch wheel installable on the GPU box (pyproject.toml + uv.lock)
The 5070Ti is Blackwell (sm_120) and needs a CUDA 12.8+ torch build (cu128). The default PyPI torch wheel is CPU-only on Windows/Linux.
- Configure uv so the CUDA torch/torchaudio build can be installed on win/linux WHILE macOS stays on the default wheel (no CUDA wheels exist for macOS -- do not break the Mac dev env). Use a platform-gated uv index/source (e.g. an `[[tool.uv.index]]` for https://download.pytorch.org/whl/cu128 marked explicit, with `[tool.uv.sources]` selecting it only for `sys_platform != 'darwin'`).
- HARD REQUIREMENT: `uv lock` and `uv sync` must still succeed on THIS macOS machine after your change (verify it). If the platform-gated index proves too fiddly to verify without a CUDA box, fall back to: leave the default wheel in pyproject, and instead document the exact one-line install command for the GPU box (in the module docstring or a README/KB note) -- but still ship the engine device logic from part 1, which is the non-negotiable deliverable.

## Constraints
- Touch ONLY: twitch_playground/speak/tts.py, twitch_playground/speak/_kokoclone.py, pyproject.toml, uv.lock. Do NOT touch settings.py/main.py/world.py (T-008 owns those) -- the engine reads SPEAK_REQUIRE_GPU/SPEAK_DEVICE from env itself.
- macOS dev must keep working (CPU or MPS); do not make GPU mandatory by default.

## Definition of done
- Engine logs its device at init and prefers CUDA>MPS>CPU.
- `SPEAK_REQUIRE_GPU=1` with no GPU raises a clear SpeakUnavailableError; default (unset) allows CPU with a visible slow-path warning.
- `uv lock`/`uv sync` succeed on macOS; the CUDA wheel path for win/linux is either configured (platform-gated) or clearly documented.
- Existing tests still pass; KB note added describing the GPU setup for the 5070Ti box.
