Research how Lobotomy Corporation constructs its character sprites from layered components.

Lobotomy Corporation is a Korean indie game by Project Moon. Its characters (Abnormalities and agents/employees) are rendered by layering multiple sprite sheets — body parts, equipment, expressions, etc — on top of each other to build a composite character.

Cover:
- How LC's sprite system works: what layers exist (body, head, limbs, equipment, etc.)
- Whether sprites are built from a shared skeleton/rig or are frame-by-frame per layer
- The file formats used (PNG sprite sheets? individual frames? spine2d/dragonbones?)
- How the game composes layers at runtime to show different equipment/states
- Any community reverse-engineering, modding tools, or sprite rippers that document the format
- How animation states are handled (idle, walk, attack, etc.) per layer
- Practical approach for a fan project to replicate this layered composition in pygame

Note: there is a Google Drive folder at https://drive.google.com/drive/folders/1WiahaZ5hBY5YZp_CFserxYf_-q65jTYZ that may contain extracted sprites. Reference this as the likely source of assets.

Deliver a research brief explaining the sprite anatomy and how to replicate it.
