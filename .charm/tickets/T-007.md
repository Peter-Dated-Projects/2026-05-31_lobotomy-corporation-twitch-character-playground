Implement the text-level profanity filter that runs on a viewer message BEFORE it goes to TTS. CPU-only, sub-millisecond, pure Python.

## Spec (already researched)
Follow `.charm/kb/swear-filter-research.md` exactly -- it contains the chosen approach, the integration snippet, and the evasion vectors. Recommendation: `better-profanity` + a Unicode normalization pre-pass.

## What to build -- `twitch_playground/speak/filter.py`
- `filter_message(text: str) -> str`: returns the message with profanity censored, per the snippet in the KB doc (NFKD homoglyph strip, spaced-letter collapse heuristic, better-profanity censor on normalized + original).
- Load the censor word list ONCE at import / first call (do not reload per message).
- Additionally cover the evasion vectors the KB flags as fixable with cheap passes: strip Unicode category Mn (combining diacritics / zalgo) after NFKD, and a pass that strips interspersed non-word chars (f.u.c.k / f-u-c-k) before checking. Keep these simple and documented.
- Expose a small hook to extend the wordlist (the KB notes `profanity.add_censor_words([...])` for Twitch-specific slang).

## Tests -- `tests/test_speak_filter.py` (pytest)
Cover: clean message passes through unchanged; basic profanity censored; leetspeak (h3ll, f*ck, @ss); spaced letters (f u c k); dotted/dashed insertion (f.u.c.k); a homoglyph/full-width case; and a false-positive guard (a normal sentence with short words like 'I am ok so go' is NOT mangled by the spaced-letter heuristic). Keep tests deterministic and fast.

## Constraints
- `better-profanity` is added to pyproject.toml by the engine ticket (T-004) -- do NOT edit pyproject.toml. If when you start it is not yet there, run `uv add better-profanity` only as a last resort and note it; normally it should already be present since this ticket depends on T-004.
- Do NOT edit settings.py or any other module. Pure, self-contained `filter.py` + its test.
- The wiring ticket will call `filter_message()` before `SpeakEngine.speak()`.

## Definition of done
- `from twitch_playground.speak.filter import filter_message` imports and works.
- `uv run pytest tests/test_speak_filter.py` passes.
- Behavior matches the KB doc's snippet + the documented extra passes.
