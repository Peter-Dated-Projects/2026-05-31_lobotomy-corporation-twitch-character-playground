Composite CharacterDefs into SpriteSets via the extraction engine, and wire it in as a selectable provider with graceful fallback. Depends on T-015 (extraction) and T-016 (char defs) — both merge first.

SPEC (authoritative): .charm/kb/architecture/character-layer-stack.md (12-layer blit order, anchor offsets, emotion states, animation-clips table), .charm/kb/conventions/character-def-format.md ('Conversion happens in lobcorp_renderer.py' = this file). CONSUME (do not modify/reimplement): assets/provider.py (SpriteSet dataclass + AssetProvider Protocol; required clips idle/walk/jump/hug), assets/sprite_extraction.py (T-015 cache API), assets/character_defs.py (CHARACTER_DEFS, assign_character, DEFAULT_CHARACTER_ID), docs/design/sidescroller-contract.md.

DELIVERABLES:
1. assets/lobcorp_renderer.py: compose_character(def, emotion, anim_frame, cache)->pygame.Surface blitting layers in the layer-stack order onto a transparent (SPRITE_W,SPRITE_H) surface using settings.LAYER_OFFSETS (center-relative, y-up negative); face layers by emotion, limb layers by anim_frame%N; skip None glasses/weapon. class LobCorpProvider(AssetProvider): get_sprite_set(character_id)->SpriteSet resolving id->CharacterDef (CHARACTER_DEFS, else assign_character/DEFAULT), building the 4 required clips per the animation-clips table (idle 2f, walk 8f default-emotion, jump stretched airborne, hug 4f), caching per character_id. Instantiate AFTER pygame.display.set_mode (same contract as PlaceholderProvider).
2. GRACEFUL FALLBACK (critical — assets are NOT in repo): if settings.ASSETS_ROOT is absent on disk or any sheet/variant load raises, fall back to PlaceholderProvider behavior and log a clear warning ONCE; app + tests must never crash from missing ~/Downloads assets.
3. Wire provider selection in main.py and dev.py: pick LobCorpProvider when assets available else PlaceholderProvider — a small make_provider() that probes settings.ASSETS_ROOT is fine. Keep changes minimal, match current construction.

DO NOT change provider.py's SpriteSet/Protocol, or touch sim/*, world.py, scene.py, hud.py.

VERIFY: `uv run --project . pytest` (full suite) stays green WITHOUT ~/Downloads assets (fallback path). If assets ARE present, run a throwaway headless smoke (SDL_VIDEODRIVER=dummy, init display, LobCorpProvider, get_sprite_set('Standard Agent'), assert each clip has (SPRITE_W,SPRITE_H) surfaces) and report real-composite-vs-fallback in your completion note. Match surrounding style.
