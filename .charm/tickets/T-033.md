PROBLEM: Characters don't move around enough -- the crowd clumps and goes static instead of wandering the wide stage. Root cause is the grouping/cohesion machinery pulling everyone together and the density-damping freezing packed characters. Goal: visibly livelier, freely-wandering crowd.

All changes are in twitch_playground/settings.py (knobs only; no sim-logic edits expected). Keep the explanatory comments accurate to the new values.

1. CROWD FLOCKING -> near-zero. Set CROWD_COH_WEIGHT = 0.0 and CROWD_ALI_WEIGHT = 0.0 so boids cohesion/alignment no longer pull characters into packs. KEEP CROWD_SEP_WEIGHT = 1.0 (separation is the anti-overlap rule -- do NOT zero it).

2. AUTONOMOUS GROUPING -> off. Set AUTONOMOUS_GROUPING = False so characters never self-form follow-clusters (explicit !follow still works).

3. GROUP EMOTION CONTAGION -> low. Lower CONTAGION_RATE from 0.6 to ~0.2 so mood no longer homogenizes the crowd into a synchronized (and often frozen) state. Leave EMOTION_DECAY_PER_SEC as-is.

4. RELAX DENSITY DAMPING (this is a direct movement killer). Raise CROWD_MIN_SPEED_SCALE (0.4 -> ~0.7 floor so packed characters keep most of their speed), lower CROWD_SLOWDOWN_PER_NEIGHBOR (0.08 -> ~0.03), and lower CROWD_IDLE_DENSITY_GAIN (0.5 -> ~0.15) so a crowded platform doesn't shuffle-and-pause.

5. BOOST WANDER VIGOR. Raise WANDER_DISPLACE (90 -> ~140) and WANDER_REORIENT_CHANCE (0.5 -> ~0.8) for more roaming, and lower IDLE_CHANCE (0.30 -> ~0.18) so they pause less.

These are starting points -- the file itself says 'tune by eye'. Pick sensible values in these directions; the named numbers are a reasonable first pass, not gospel. Do NOT touch JUMP physics, the LobCorp asset/compositing block, or lifecycle knobs. Verify the module still imports cleanly (GRID_CELL derives from the radii, which you are not changing, so it stays valid).
